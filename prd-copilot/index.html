核心判断
通用文档 AI 强在写作和协作，但通常默认用户已经知道自己要做什么。本工具选择更前置的位置：先判断需求成熟度，再把模糊想法翻译成可评审 PRD。

1输入模糊需求

自定义输入

建议输入具体业务场景，比如创作者成长、内容安全、AI 数据复盘。

2需求澄清 Copilot

等待分析
输入需求后，系统会判断需求成熟度，列出缺失信息、建议追问问题，并生成一张可进入 PRD 的需求卡。

3PRD Builder

PRD 会在这里生成。它会包含背景、目标用户、用户场景、产品目标、功能范围、流程、指标、风险和验收标准。
这个 PRD 草稿对你有帮助吗？
用于判断澄清问题和 PRD 输出是否能直接进入评审或继续改稿。
哪里不够好？
下一步动作 把生成的 PRD 作为需求评审草稿：它把运营洞察、实验结论和用户问题翻译成可讨论、可排期、可验收的产品需求。 返回合集 →
核心指标 需求澄清补全率 · PRD 评审通过率 · 人工修改比例 · 需求返工率。
